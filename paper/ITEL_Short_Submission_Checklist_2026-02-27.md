# ITEL Short Submission Checklist (as of 2026-02-27)

Use this right before submission.

## 1. Track And Format
- [ ] Selected `Short Paper` track in the submission system.
- [ ] Manuscript is in English.
- [ ] First submission length matches short-paper expectation (target: 4 pages).
- [ ] Source file is prepared from the official ITEL paper format/template.
- [ ] PDF output has no broken fonts, unreadable figures, or overflow text.

## 2. Scope Fit
- [ ] Paper clearly addresses Technology-Enhanced Learning context.
- [ ] Contribution is stated as practical and educationally meaningful.
- [ ] Research questions are explicit and aligned with method/results.

## 3. Method Transparency
- [ ] Participants, grade level, and classroom context are reported.
- [ ] PRE/POST instrument structure is fully described (item counts and scale).
- [ ] Analysis method is explicit (paired test type, effect size, qualitative coding process).
- [ ] Exclusion criteria and final analyzed sample size are stated.

## 4. Ethics And Consent
- [ ] Voluntary participation and no-penalty policy are explicitly written.
- [ ] Consent for research use is separated from quote consent in the method text.
- [ ] Anonymization/de-identification process is described.
- [ ] Institutional policy status is stated (approval/exempt/not-required determination).

## 5. Writing Quality
- [ ] Abstract includes context, method, key numbers, and implication.
- [ ] Claims avoid overstatement (no causal language without design support).
- [ ] Limitations section is concrete (sample bias, self-report, duration limits).
- [ ] Grammar and terminology are consistent across the manuscript.

## 6. Figure And Table Quality
- [ ] One compact system-flow figure is included (recommended).
- [ ] One table for participation/consent and one for pre/post comparison are included.
- [ ] Captions are self-contained and interpretable without main text.

## 7. Submission Operations
- [ ] Author metadata in the system matches manuscript exactly.
- [ ] Required declarations (conflict of interest, ethics statements) are completed.
- [ ] Co-authors confirmed the final version before upload.
- [ ] Submission PDF uploaded and confirmation email received.

## 8. Suggested Final Pass (15 minutes)
1. Read title, abstract, and conclusion back-to-back for consistency.
2. Verify all `[FILL]` placeholders are removed.
3. Re-check participant counts across abstract, table, and method.
4. Confirm p-values and effect sizes are internally consistent.
5. Export final PDF again and perform one visual QA pass.
