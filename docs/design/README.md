# 設計ドキュメント

## 概要

本ディレクトリには、SFプロトタイピング教育プラットフォームの設計ドキュメントが含まれています。

---

## ドキュメント一覧

### 1. [システムアーキテクチャ設計書](./system_architecture.md)

システム全体のアーキテクチャ設計を定義します。

**内容**:
- アーキテクチャ概要
- 技術スタック
- ディレクトリ構造
- データフロー
- セキュリティアーキテクチャ
- パフォーマンス最適化戦略
- エラーハンドリング戦略
- モニタリング・ログ戦略
- デプロイメントアーキテクチャ

---

### 2. [データベース設計書](./database_schema.md)

データベーススキーマとPrismaスキーマを定義します。

**内容**:
- ER図（概念モデル）
- Prismaスキーマ定義
- インデックス戦略
- データ型の詳細
- リレーション設計
- マイグレーション戦略
- シードデータ
- パフォーマンス考慮事項

---

### 3. [API設計書](./api_specification.md)

RESTful APIとSSEの仕様を定義します。

**内容**:

- セッション管理API
- Big Five診断API
- テーマ・質問API
- 回答API
- 学生管理API
- 可視化API
- 統計API
- Server-Sent Events (SSE)
- エクスポートAPI
- エラーハンドリング
- レート制限

---

### 4. [デザイントークン定義書](./design_tokens.md)

デザインシステムのトークン定義を記載します。

**内容**:
- カラーパレット（生徒側・管理者側）
- タイポグラフィ
- スペーシングシステム
- シャドウシステム
- ボーダー・角丸
- アニメーション
- コンポーネント状態
- Tailwind CSS設定
- TypeScript型定義
- Figmaとの連携

---

### 5. [UI/UX設計書](./ui_ux_design.md)

画面設計とコンポーネント設計を定義します。

**内容**:
- 画面遷移図
- 基本UIコンポーネント（ボタン、入力、カード、テーブル等）
- 生徒向け画面設計（Big Five診断、テーマ選択、質問回答、意見マッピング）
- 管理者向け画面設計（ダッシュボード、結果分析）
- レスポンシブデザイン
- アクセシビリティ
- アニメーション・トランジション

---

## 設計の流れ

1. **システムアーキテクチャ設計**: 全体構造を決定
2. **データベース設計**: データモデルを定義
3. **API設計**: エンドポイント仕様を定義
4. **デザイントークン定義**: デザインシステムの基盤を構築
5. **UI/UX設計**: 画面とコンポーネントを設計
